Le problème The problem

Les PDFs sont partout — contrats, factures, rapports, formulaires scannés. Mais ils sont structurellement hétérogènes : certains sont du texte brut, d'autres des tableaux complexes, d'autres encore des images scannées illisibles par les outils classiques. Extraire les bonnes informations au bon endroit de façon fiable et automatique reste un problème non résolu pour la plupart des entreprises. PDFs are everywhere — contracts, invoices, reports, scanned forms. But they are structurally heterogeneous: some are plain text, others complex tables, others scanned images unreadable by classic tools. Extracting the right information reliably and automatically remains an unsolved problem for most organisations.

La solution The solution

J'ai conçu et développé un agent IA autonome basé sur LangGraph — un graphe d'état à 5 nœuds qui orchestre 4 extracteurs spécialisés. L'agent analyse d'abord le contenu, choisit dynamiquement la meilleure stratégie d'extraction, valide les données extraites via Pydantic v2, et retourne un résultat structuré propre. Le tout est exposé via une API FastAPI avec interface web, tracé avec MLflow, containerisé avec Docker et déployé en production sur Render via GitHub Actions. I designed and built an autonomous AI agent based on LangGraph — a 5-node state graph orchestrating 4 specialised extractors. The agent analyses the content first, dynamically picks the best extraction strategy, validates the output via Pydantic v2, and returns clean structured data. Everything is exposed via a FastAPI with web UI, tracked with MLflow, containerised with Docker and deployed to production on Render via GitHub Actions.
