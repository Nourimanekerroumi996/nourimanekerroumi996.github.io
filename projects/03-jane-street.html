Le défi The challenge

La compétition Jane Street propose un dataset massif (~12 GB) de données financières anonymisées. L'objectif : prédire les mouvements de marché futurs en exploitant des patterns temporels complexes et des features hautement corrélées. Les contraintes : données temporelles avec dépendances non-triviales, 130+ features anonymisées, déséquilibre de classes important, limite de mémoire stricte, et scoring en temps réel sur données cachées. The Jane Street competition provides a massive dataset (~12 GB) of anonymised financial data. The goal: predict future market movements by exploiting complex temporal patterns and highly correlated features. Constraints: temporal data with non-trivial dependencies, 130+ anonymised features, significant class imbalance, strict memory limits, and real-time scoring on hidden data.

L'approche The approach

J'ai construit un pipeline de feature engineering + XGBoost optimisé pour maximiser le ROC AUC tout en respectant les contraintes de mémoire. L'approche : création de features temporelles (lags, rolling statistics, interactions), sélection basée sur l'importance SHAP, XGBoost avec hyperparamètres optimisés via Optuna, validation temporelle (time-series split), et gestion mémoire stricte (chunking + garbage collection). I built a feature engineering + XGBoost pipeline optimised to maximise ROC AUC while respecting memory constraints. The approach: temporal feature creation (lags, rolling statistics, interactions), SHAP-based feature selection, XGBoost with Optuna-optimised hyperparameters, temporal validation (time-series split), and strict memory management (chunking + garbage collection).
