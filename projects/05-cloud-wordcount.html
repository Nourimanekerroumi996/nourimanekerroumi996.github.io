Le défi The challenge

Projet académique à Cranfield University : construire un système distribué capable de traiter des datasets textuels massifs en parallèle. Le challenge : diviser un fichier de 1+ MB en 1000+ chunks, distribuer les tâches via SQS, orchestrer 4 instances EC2 workers, et agréger les résultats — le tout avec automation complète, fault tolerance, et optimisation des coûts AWS. Academic project at Cranfield University: build a distributed system capable of processing massive text datasets in parallel. The challenge: split a 1+ MB file into 1000+ chunks, distribute tasks via SQS, orchestrate 4 EC2 worker instances, and aggregate results — all with full automation, fault tolerance, and AWS cost optimization.

L'approche The approach

J'ai construit un pipeline cloud-native entièrement automatisé : chunking dynamique du fichier source (chunk size optimal = 1 MB via performance analysis), upload vers S3, queuing des tâches dans SQS, déploiement EC2 avec scripts workers en SSH/SCP, processing parallèle avec boto3, et aggregation finale. J'ai implémenté un système de refresh automatique des credentials AWS Learner Lab (expiration 4h) via Paramiko, monitoring CloudWatch avec alarmes custom, et cost analysis comparatif sur différentes configurations (chunk size × instances). I built a fully automated cloud-native pipeline: dynamic file chunking (optimal chunk size = 1 MB via performance analysis), S3 upload, SQS task queuing, EC2 deployment with SSH/SCP worker scripts, parallel processing with boto3, and final aggregation. I implemented an automated AWS Learner Lab credentials refresh system (4h expiration) via Paramiko, CloudWatch monitoring with custom alarms, and comparative cost analysis across different configurations (chunk size × instances).
